## Описание

Проект состоит из backend API `MemesApi` и frontend-страницы в папке `frontend`. Страница загружает мемы с сервера, добавляет новые карточки и удаляет их через `fetch`.

## Запуск

Backend:

```bash
cd MemesApi
dotnet run
```

Frontend:

Открыть `frontend/index.html` через Live Server в VS Code.

Адрес API указан в `frontend/js/api.js`:

```text
http://localhost:5110/api/memes
```

## Маршруты API

| Метод | URL | Описание |
| --- | --- | --- |
| GET | `/api/memes` | Получить все мемы |
| GET | `/api/memes/{id}` | Получить мем по id |
| POST | `/api/memes` | Добавить мем |
| DELETE | `/api/memes/{id}` | Удалить мем |

## Что изучили

| Концепция | Где используется |
| --- | --- |
| `fetch(url)` | GET-запрос к API |
| `fetch(url, { method, headers, body })` | POST и DELETE запросы |
| `response.ok` | Проверка успешности ответа |
| `response.json()` | Чтение JSON из ответа |
| `JSON.stringify(...)` | Объект JS → JSON-строка для отправки |
| `async / await` | Ожидание ответа от сервера |
| `try / catch / finally` | Обработка ошибок при `fetch` |
| CORS | Разрешение запросов между разными портами |
| `AddCors + UseCors` | Включение CORS в ASP.NET Core |
| `API_URL` | Адрес сервера в одном месте |
| `api.js` | Все сетевые запросы отдельно |
| `button.disabled = true` | Блокировка кнопки во время запроса |
| DevTools Network | Просмотр запросов и ответов |

## Выводы

1. `fetch` позволяет браузеру отправлять GET, POST и DELETE запросы к API.
2. CORS нужен, чтобы frontend на одном порту мог обращаться к backend на другом.
3. `api.js` удобно хранит сетевую логику отдельно от логики интерфейса.
4. `try/catch/finally` помогает показывать ошибки и возвращать кнопку в рабочее состояние.
5. Счётчик мемов обновляется при загрузке, добавлении и удалении карточек.
